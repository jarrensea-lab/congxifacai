# 恭喜发财 V7 路线图

> 最后更新: 2026-07-01 | 当前 feature: v7.4.0-dev

## v7.4.0-dev：量化生命周期闭环底座

目标：把恭喜发财从“报告型 AI 助手”升级为未来量化管理底座。当前先把架构、数据、策略、监控、预警闭环跑通；小账户阶段以盈利验证为第一目标，允许激进试错，但每笔试错必须有明确亏损上限。

### P0

- [x] 候选标的池 `candidate_pool`：池外发现、入池观察、升级建仓、剔除出池。
- [x] 持仓管理池 `position_watch`：买入理由、止损、止盈、目标价基础字段。
- [x] 定时任务扫描：盘前入池，午盘/午后扫描候选池与持仓池，触发机会/风险后推送飞书。
- [x] 飞书动作提醒：区分可试仓、禁止追、止损、止盈。
- [ ] Sentinel/Serenity 入池：研究包必须转为候选池事件，不能只展示在报告里。

### P1

- [ ] 池生命周期日报：新入池、继续观察、升级、出池、错失复盘。
- [ ] 小账户激进参数：单笔 5%-15% 试仓，单笔硬亏损 1.5%-3%。
- [ ] 告警去重与升级：同一标的同类提醒冷却，风险升级立即打破冷却。

### v7.5 / v8 预留

- [ ] 分钟级轮询扫描器。
- [ ] 秒级行情通道。
- [ ] 回测驱动的策略参数校准。
- [ ] 半自动/自动下单前的权限与安全边界。
- [ ] 多账户与资金分层路由。
