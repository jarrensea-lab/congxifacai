# v7.4.0-dev 生产闭环方案

日期：2026-07-01
分支：`codex/quant-lifecycle-v7-4`
版本：`v7.4.0-dev`

## 结论

当前不做分钟级和秒级能力。v7.4 首轮目标是把架构、数据、策略、监控、预警跑通，形成可以每天执行、可以复盘、可以逐步升级的生产闭环。

## 当前没有必须用户补充的信息

已具备继续推进的必要条件：

- 云模型 Key 已由环境配置承载。
- 飞书 Webhook 已由环境配置承载。
- 当前账户状态已明确：空仓、小账户、现金为主。
- 当前策略方向已明确：小账户以盈利验证为目标，允许激进试错，但必须有亏损边界。

后续仅在以下事项发生时需要用户再确认：

- 接入真实自动下单。
- 调整单票仓位、止损比例、最大回撤等资金边界。
- 新增或替换私密凭证。
- 将研究结论直接转成买入动作，而不是先入池观察。

## 最优解

采用“生产候选池 + 持仓监控池 + 定时/手动扫描 + 飞书动作提醒”的渐进方案。

### 1. 生产候选池

盘前 AI 推荐、Sentinel/Serenity 研究包、盘中新闻线索、用户手动关注都统一进入 `candidate_pool`。

候选池外标的：

- 数据充分且可执行：进入可试仓提醒。
- 数据不足但逻辑未证伪：入池观察。
- 数据不足且逻辑弱：不入池或直接丢弃。

候选池内标的：

- 符合涨幅、量比、成交额、可买性规则：升级为可试仓提醒。
- 数据不足但逻辑仍成立：继续观察。
- 涨停或接近涨停：禁止追高，次日观察。
- 逻辑失效：出池。

### 2. 持仓监控池

所有持仓必须进入 `position_watch`，记录：

- 买入理由。
- 成本价。
- 止损价。
- 目标价。
- 失效条件。
- 复核时间。

午后风控必须读取 `position_watch`，不能只按成本价做泛化风险提示。

### 3. 触发方式

当前阶段使用：

- 盘前任务：推荐入池。
- 午盘任务：扫描候选池与持仓池。
- 午后任务：扫描候选池与持仓池，触发风险/机会提醒。
- 收盘任务：复盘状态迁移。
- 手动脚本/API：临时扫描。

分钟级轮询和秒级行情后置到 v7.5/v8。

### 4. 飞书提醒

飞书提醒必须明确动作，而不是泛泛观望：

- 可试仓：允许人工确认后试仓。
- 禁止追：涨停或过热，不能追。
- 止损：必须处理。
- 止盈：考虑分批止盈或移动止损。
- 逻辑失效：减仓或出池。

### 5. Sentinel/Serenity 边界

Sentinel/Serenity 仍然是研究层，不直接下单。

正确接入方式：

- 研究主题、候选公司、证据摘要转为候选池事件。
- 候选池扫描决定继续观察、可试仓、禁止追、出池。
- 飞书只推生产侧动作结论，完整研究归档保留本地。

## 验证口径

- 单元测试必须覆盖候选池和持仓池核心状态迁移。
- 空仓午盘/午后不能只推“空仓观望”，必须扫描候选池。
- 涨停附近标的必须提示“禁止追高”，不能静默。
- 可一手执行且放量上涨标的必须触发可试仓提醒。
- 持仓跌破止损必须生成 high alert。
- 持仓触及目标必须生成止盈提醒。
- 飞书卡片不能因为 `medium` / `mid` 字段不一致漏告警。

## 风险边界

- 当前系统只做提醒，不做自动下单。
- 所有买入/卖出仍需人工确认。
- 小账户可以激进，但每笔都必须有止损价和仓位上限。
- 数据源异常时只能降级为“数据不可用”，不能强行输出买入建议。
