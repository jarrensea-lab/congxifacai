## 2. 目标运行节奏

### 2.1 主报告

主报告名称建议：

```text
YYYY-MM-DD_次日投资策略主报告.md
```

生成时间：

- 交易日盘后：建议 `20:30` 生成，服务下一交易日。
- 周日晚上：建议 `20:30` 生成，若下一个交易日是周一，则服务周一。
- 节假日前夜：仅当下一个自然可交易日存在且在合理窗口内时生成，避免长假期间每天重复。

报告服务对象：

- 次日盘前决策。
- 用户人工复核。
- 盘前短校准的基础输入。

### 2.2 盘前短策略校准

报告名称建议：

```text
YYYY-MM-DD_盘前短策略校准.md
```

生成时间：

- 交易日 `08:50`。

内容边界：

- 不重写主报告。
- 只检查隔夜/早间增量变化。
- 只回答：主报告是否仍然有效、风险等级是否变化、是否有新禁止条件或新观察项。

### 2.3 盘中与收盘报告

- 盘中分析：保留 `11:35` 与 `14:00` 两个节点。
- 收盘复盘：保留 `15:05`。
- 系统状态：保留每日归档，优先用于排查自动化运行问题。

## 3. 主报告目标结构

主报告必须从“看法集合”升级为“次日策略剧本”。

建议结构：

```text
1. 明日总策略
   - 交易 / 不交易 / 只观察
   - 风险等级
   - 仓位上限
   - 禁止动作

2. 账户约束
   - 可用现金
   - 当前持仓
   - 单笔预算
   - 可买一手价格上限
   - 现金安全垫

3. 数据源审计
   - 行情数据状态
   - Horizon/Tushare 新闻覆盖
   - Sentinel 输入状态
   - DeepSeek/Qwen 状态
   - 缺失或降级项

4. 市场状态
   - 指数趋势
   - 情绪/资金/风险事件
   - 关键支撑压力

5. Sentinel 研究输入
   - 今日主题雷达
   - 高频新闻摘要
   - 风险主题
   - 候选池变化
   - 不可交易但需跟踪的研究结论

6. 四人辩论矩阵
   - 猎手：短线技术观点
   - 账房：基本面/估值观点
   - 守夜人：风控与生存纪律
   - Serenity：产业链瓶颈观点
   - 每个角色必须列出观点、证据、反对意见、执行可行性

7. 裁判裁决
   - 采用哪些观点
   - 否决哪些观点
   - 为什么
   - 对明日策略的最终约束

8. 明日执行剧本
   - 触发条件
   - 禁止条件
   - 观察名单
   - 买入前置条件
   - 退出/止损条件

9. 后续验证
   - 本报告哪些判断进入 Sentinel 预测留痕
   - 1/3/5/20 日后如何回看
```

验收标准：

- 用户只看第 1 节和第 8 节，也能知道明天怎么做。
- 用户看第 3 至第 7 节，能知道策略从哪些证据推导出来。
- 每条可执行建议都必须通过账户约束和一手金额校验。
- 任何研究性结论都必须明确标记为“研究输入”，不能伪装成交易指令。

## 7. 调度治理线

### 7.1 主调度原则

推荐主路径：

```text
launchd 常驻 FastAPI
→ FastAPI APScheduler 负责交易时段任务
→ 手动脚本作为兜底
```

保留兜底路径：

```text
scripts/daily_report.py
scripts/run_premarket.py
```

验收：

- 系统重启后，launchd 能自动拉起恭喜发财服务。
- FastAPI 启动后，APScheduler 能看到所有任务。
- 手动脚本仍能独立生成报告。

### 7.2 调度时间建议

```text
周日 20:30       主报告：服务周一
交易日 08:50     盘前短策略校准
交易日 11:35     盘中分析
交易日 14:00     午后风控
交易日 15:05     收盘复盘
交易日 20:30     主报告：服务下一交易日
每日 21:00       Sentinel 绩效回看与归档
```

注意：

- 周日主报告不能用简单 `is_trading_day()` 判断，因为周日本身不是交易日。
- 应使用“下一个交易日是否为周一/是否存在下一交易日”的逻辑。
- 节假日必须使用 A 股交易日历，不能只靠 `mon-fri`。
- Sentinel 绩效回看可以每日触发，但只有存在到期预测、当天主报告或当天执行记录时才生成新报告，避免空报告刷屏。
