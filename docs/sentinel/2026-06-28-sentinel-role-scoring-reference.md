# Sentinel 角色评分参考

> 日期：2026-06-28
> 用途：作为 Sentinel 角色预测与投资建议绩效评估的评分口径参考
> 边界：本文件用于评估辩论系统质量，不构成投资建议，不自动修改角色权重或 prompt。

## 1. 核心原则

角色评分不是简单判断“谁对谁错”。

四名辩论员和裁判的职责不同、视角不同、风险偏好也不同，所以评分要看：

- 这个角色是否准确。
- 这个角色是否赚钱或减少亏损。
- 这个角色是否履行了自己的职能。
- 这个角色的判断为什么不够精准。
- 裁判是否正确综合了不同角色意见，并给出最终有效策略。

一个激进角色可以承担更高波动，但不能没有证据地乱推荐。
一个保守角色可以错过部分收益，但不能长期把所有机会都否掉。
一个研究角色可以不直接判断买卖，但必须提供可验证的产业链和事实证据。

## 2. 一级评分维度

### 2.1 准确性与收益评估

这一维度回答：

- 谁猜得准。
- 谁猜得不准。
- 谁让我赚钱。
- 谁害我亏钱。
- 谁帮我避免亏损。
- 判断是否足够精准。
- 不够精准的原因是什么。

建议观察项：

- 方向判断是否命中。
- 标的表现是否符合预期。
- 板块或主题表现是否符合预期。
- 收益是否跑赢合理基准。
- 风险提示是否提前出现。
- 错误来自信息不足、逻辑错误、过度乐观、过度保守，还是市场突发事件。

注意：

- 不能只看涨跌。
- 不能只看是否跑赢基准。
- 不能只看风险是否降低。
- 必须结合角色当时的职责和预测周期评价。

### 2.2 执行纪律归因

这一维度回答：

- 角色给出了合理建议，但用户没有执行，是否应该扣角色分。
- 用户执行偏离建议，结果亏损，是否应该扣角色分。
- 角色建议本身是否清晰、合理、可执行。
- 用户是否按照建议的价格、仓位、时间和风险条件执行。

基本原则：

- 角色评分只评价角色当时的判断质量和建议质量。
- 如果角色给出合理建议，但用户没有执行，不扣角色分。
- 如果用户没有按建议执行，执行结果不能直接归因给角色。
- 用户执行纪律要单独记录和评分。
- 裁判最终策略也要区分“策略质量”和“执行质量”。

建议记录项：

- `advice_given`：是否给出明确建议。
- `advice_quality`：建议是否合理、完整、可执行。
- `user_executed`：用户是否执行。
- `execution_match`：执行是否符合建议。
- `execution_deviation_reason`：未执行或偏离原因。
- `role_score_impact`：是否影响角色评分。
- `discipline_score_impact`：是否影响用户执行纪律评分。

示例：

- 猎手建议“只在放量突破后小仓位观察”，用户追高重仓买入后亏损：不应主要扣猎手分，应记录用户执行纪律偏离。
- 守夜人建议“风险过高，先观望”，用户仍买入后亏损：守夜人应加分，用户执行纪律扣分。
- 裁判给出模糊建议，导致无法判断怎么执行：裁判建议质量扣分，不归因给用户执行纪律。

### 2.3 职责履行情况

这一维度回答：

- 辩手是否按照自己的职能在行事。
- 是否越界给出不属于自己职责的建议。
- 是否胡乱给出建议。
- 是否没有调查研究。
- 是否调查得不深入。
- 是否忽略了明显反证。

建议观察项：

- 是否引用了足够证据。
- 是否说明了判断依据。
- 是否区分事实、推断和待核验。
- 是否说明了不确定性。
- 是否识别了自己角色应该关注的风险。
- 是否在证据不足时降低结论强度。

### 2.4 裁判评分

裁判必须单独评分。

裁判不是第五名辩手，而是综合四位辩手意见后的最终判断者。裁判的评分权重最高，因为用户最终看到并执行的是裁判给出的综合策略。

裁判评分重点：

- 是否正确理解四位辩手的意见。
- 是否识别出谁的证据更强。
- 是否没有被单一角色带偏。
- 是否正确处理风险和收益的冲突。
- 是否给出清晰、可执行、符合账户约束的策略。
- 最终策略是否让用户获得盈利，或减少亏损。
- 如果策略失败，失败原因是否可解释。

裁判的最终标准：

- 策略有没有帮助账户。
- 策略有没有尊重风控。
- 策略有没有正确综合证据。
- 策略有没有避免明显不该犯的错误。
- 策略质量和用户执行质量是否被分开归因。

## 3. 默认评分参数

第一版采用以下默认参数。

### 3.1 评分周期

- 1 日
- 3 日
- 5 日
- 20 日

用途：

- 1 日和 3 日偏短线验证。
- 5 日观察短中期延续。
- 20 日观察中低频判断和产业链观点。

### 3.2 对标基准

- 沪深 300
- 中证 2000
- 对应行业或主题指数

说明：

- 大盘判断优先对标沪深 300。
- 小盘和题材判断优先对标中证 2000 或对应主题指数。
- 个股建议优先看相对所属行业或主题的表现。

### 3.3 角色评分权重

四名辩论员默认权重：

- 结果表现：40%
- 职责履行：35%
- 风险控制：15%
- 解释质量：10%

裁判单独权重：

- 最终账户影响：50%
- 综合判断质量：30%
- 风控遵守：20%

执行纪律单独评分，不混入角色评分。

### 3.4 报告频率

- 每日：简版摘要。
- 每周：完整角色评分报告。
- 每月：角色权重和 prompt 复核建议。

### 3.5 调整权限

Sentinel 只允许提出：

- 提高权重
- 降低权重
- 观察
- 复核 prompt

Sentinel 不自动修改权重，不自动修改 prompt，不自动交易。

## 4. 角色专属评价口径

### 4.1 猎手

猎手偏短线和进攻。

重点看：

- 短线方向是否敏锐。
- 入场时点判断是否有效。
- 技术信号是否有证据。
- 是否过度追高。
- 是否忽略流动性和波动风险。

不能苛责：

- 不要求猎手承担完整基本面判断。
- 不要求猎手长期价值判断一定准确。

必须扣分：

- 没有技术证据就强推。
- 只看热点不看风险。
- 把短线机会包装成长线确定性。

### 4.2 账房

账房偏基本面、估值和中低频。

重点看：

- 财务和估值判断是否扎实。
- 是否识别估值过高或质量不足。
- 是否能保护账户不追高烂资产。
- 中低频判断是否稳定。

不能苛责：

- 不要求账房捕捉每一次短线题材爆发。
- 不要求账房在情绪行情里总是最赚钱。

必须扣分：

- 没有财务证据就判断便宜或贵。
- 忽略负债、现金流、盈利质量。
- 用过时数据做强结论。

### 4.3 守夜人

守夜人偏风险、回撤和不可操作条件。

重点看：

- 风险提示是否提前。
- 是否识别重大亏损风险。
- 是否保护账户免受大回撤。
- 是否正确提醒仓位、止损、流动性和黑名单。

不能苛责：

- 守夜人偏保守是职责的一部分。
- 它错过上涨机会不一定等于失败。

必须扣分：

- 风险证据不足却一律否决。
- 长期过度保守导致系统失去机会识别能力。
- 漏掉明显风险事件。

### 4.4 Serenity

Serenity 偏产业链、瓶颈、供需和研究证据。

重点看：

- 产业链判断是否有事实依据。
- 是否识别真正瓶颈，而不是泛泛讲题材。
- 是否能提出需要核验的公司、环节和风险。
- 是否区分产业趋势与个股收益。

不能苛责：

- 不要求 Serenity 直接给买卖点。
- 不要求产业链判断立刻兑现到股价。

必须扣分：

- 没有资料就讲故事。
- 把主题热度等同于确定收益。
- 忽略财务、公告和行情反证。

### 4.5 裁判

裁判偏综合决策。

重点看：

- 是否吸收四位辩手的有效信息。
- 是否正确裁剪无效、越界或证据弱的观点。
- 是否给出最终策略，并说明理由。
- 是否符合账户现金、持仓和风险约束。
- 策略执行后是否赚钱、少亏或改善风险。

必须扣分：

- 机械平均四位辩手意见。
- 被最激进或最保守角色带偏。
- 给出模糊、无法执行的结论。
- 忽略用户真实账户约束。

## 5. 建议动作边界

Sentinel 可以给出以下建议：

- `提高权重`
- `降低权重`
- `观察`
- `复核 prompt`

Sentinel 不可以：

- 自动修改角色权重。
- 自动修改 prompt。
- 自动替代裁判。
- 自动触发交易。

任何角色调整都必须由用户确认后再进入配置。

## 6. 推荐评分输出结构

每次评分建议输出四类内容：

1. 结果表现：这次判断带来了什么结果。
2. 职责履行：角色是否做好了自己的本职工作。
3. 误差原因：如果不准，为什么不准。
4. 调整建议：提高权重、降低权重、观察或复核 prompt。

示例结构：

```json
{
  "role": "hunter",
  "period": "5d",
  "accuracy": {
    "direction_hit": true,
    "return_error": 0.018,
    "reason": "方向判断正确，但高估了反弹幅度"
  },
  "duty_fulfillment": {
    "score": 82,
    "reason": "技术信号清楚，但对成交量持续性核验不足"
  },
  "outcome": {
    "paper_advice": true,
    "executed_by_user": false,
    "account_impact": "not_applicable",
    "execution_discipline": "not_executed_by_user",
    "role_score_impact": "no_penalty"
  },
  "recommendation": {
    "action": "watch_role",
    "reason": "短线敏感度有效，但需要观察高置信度判断的稳定性"
  }
}
```

## 7. 评分解释原则

评分报告必须用人能看懂的话解释：

- 准在哪里。
- 错在哪里。
- 是职责内的合理偏差，还是职责失守。
- 是市场突发导致，还是分析质量不足。
- 是角色建议质量问题，还是用户执行纪律偏离。
- 下一步应该提高权重、降低权重、观察，还是复核 prompt。

评分报告不能只给数字。
